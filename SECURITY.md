# Security Policy

## Supported Versions

Security fixes are provided for the latest code on the default branch.

| Version | Supported |
| ------- | --------- |
| 0.2.x   | Yes       |
| < 0.2   | No        |

## Reporting a Vulnerability

Please do not open a public issue for security reports.

Use GitHub private vulnerability reporting for this repository. If that option is not available, contact the maintainers through the repository contact channel and include:

1. A clear description of the issue.
2. Steps to reproduce the issue.
3. Any proof of concept or logs needed to validate the report.
4. Potential impact and affected components.

## Response Process

1. We acknowledge receipt as soon as possible.
2. We validate and triage the report.
3. We prepare and release a fix.
4. We coordinate disclosure timing with the reporter.
